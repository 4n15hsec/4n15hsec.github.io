Ignite (Rewritten)

Target IP: 10.10.253.193
Challenge Description: Root the box!

Reconnaissance


Performing a TCP port scan using the command sudo nmap -sS 10.10.253.193 -p- returns the result shown above. There is only one TCP port open on the target machine: HTTP on port 80. Time to perform an aggressive scan to identify its application.


I ran an aggressive scan against the HTTP on port 80 by using the command sudo nmap -sV -A 10.10.253.193 -p 80 and obtained the result shown above. According to the scan, the target machine is running Fuel CMS on this port. Also, the robots.txt seems to contain the entry /fuel which is the login webpage page of the CMS.

Enumeration

Port 80: HTTP

The default webpage of Fuel CMS is returned to me for this application. I notice the target machine is running the version 1.4. I viewed the source-code of the webpage but I did not find anything important & useful. Time to access the login webpage.


I browsed to http://10.10.253.193/fuel and the login webpage shown above was presented to me. From past experience, I know the default credentials of Fuel CMS is admin:admin. Time to try it.


And bingo. I managed to successfully login as the user admin by using the credentials admin:admin. Now I have administrator access on the web application. I notice this user has the super admin flag enabled too, which is useful when uploading malicious files to the web server; for example, a web shell. One possible attack vector is to upload a PHP webshell and then obtain a reverse shell connection using it. However, Fuel CMS version <= 1.4.1 are vulnerable to Remote Code Execution (RCE). This vulnerability has the CVE ID of CVE-2018-16763. Since the target machine is running a vulnerable version, I can just exploit it.

Privilege Escalation

Vertical Privilege Escalation: www-data user to root user via Credential Hunting


After plenty of manual enumeration, I found an interesting called database.php located at /var/www/html/fuel/application/config, as shown above. This file contains the credentials root:mememe. Using this credentials, I managed to fire up MySQL using the command mysql -u root -p mememe, but I did not find anything important. Can I switch user instead?


Using the command su root and the password mememe, I successfully managed to switch user to root as shown above. Now I have root access on the target machine with the highest privileges. GG.
